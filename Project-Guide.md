**Property Recommender App – Project Guide**

1. **Project Goal**

   * Build a property recommender app that recommends properties and compounds based on user natural language semantic and keyword search with filters.
   * The app will suggest properties with their data and pass them to an LLM for a more guided response.
   * The LLM will act as a property consultant or broker to guide the user to the best recommendation.
   * This will help the customer build intuition about what they want and compare options with the LLM to find the best recommendation.

2. **Data Collection**

   * Ingest or collect property data from Nawy because the app is being built for them.
   * Perform this data collection using scraping.

3. **Data Processing**

   * Clean the collected data.
   * Organize the cleaned data.

4. **Recommender System Development**

   * Build a recommender system using:

     * Vector search for semantic search.
     * BM25 for keyword search.
   * Add filters to refine the recommendations.

5. **LLM-Based Analysis and Guidance**

   * Take the recommendations generated by the recommender system and pass them to an LLM.
   * The LLM will compare the recommendations and determine the best option based on analysis.

6. **User Action**

   * The user may take an action to contact the company for more details and proceed with buying the property.
